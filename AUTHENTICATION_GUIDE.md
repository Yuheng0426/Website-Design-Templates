# Authentication Guide

This template includes a polished sign-in page, but it does not include a real login system yet. That is intentional. Authentication should be connected carefully because it handles private user identity, secrets, sessions, and account access.

Use this guide when you are ready to connect Google sign-in or another provider.

## What The Template Already Includes

- `Web_Template/Views/Home/Login.cshtml`: the visual sign-in page.
- `Web_Template/wwwroot/css/site.css`: the login page layout and provider button styles.
- `Web_Template/wwwroot/js/site.js`: demo-only click handling and translated login text.
- `docs/login.html`: the static GitHub Pages preview, generated by `scripts/build-static-demo.js`.

The current provider buttons use `data-auth-demo`. They show a demo message and do not send account data anywhere.

## Recommended Beginner Path

1. Keep the current sign-in page as a visual template.
2. Learn ASP.NET Core Identity before storing users.
3. Add Google only after the local email login works.
4. Store secrets outside source code.
5. Test redirects locally before deploying.
6. Regenerate the static demo after visual changes.

## Connect Google Sign-In With ASP.NET Core

Follow the official Microsoft guide first:

[Google external login setup in ASP.NET Core](https://learn.microsoft.com/en-us/aspnet/core/security/authentication/social/google-logins)

The high-level setup is:

1. Create or enable ASP.NET Core Identity in the project.
2. Create a Google Cloud project.
3. Configure the OAuth consent screen.
4. Create an OAuth client ID for a web application.
5. Add authorized redirect URIs.
6. Add the Google authentication package.
7. Store the Google client ID and client secret safely.
8. Configure the authentication pipeline in `Program.cs`.
9. Replace the demo Google button with a real external login route.

## Google Redirect URI

Google must know where to send the user after sign-in. For an ASP.NET Core MVC app using the Google authentication handler, the local redirect URI is usually:

```text
https://localhost:5001/signin-google
```

Your local port may be different. Use the HTTPS port shown by your app.

For production, add your deployed domain too:

```text
https://your-domain.com/signin-google
```

Google documents redirect URIs in its OAuth web server guide:

[Using OAuth 2.0 for Web Server Applications](https://developers.google.com/identity/protocols/oauth2/web-server)

## Store Secrets Safely

Never commit a client secret to GitHub.

For local development, use user secrets:

```powershell
dotnet user-secrets init --project .\Web_Template\Web_Template.csproj
dotnet user-secrets set "Authentication:Google:ClientId" "YOUR_CLIENT_ID" --project .\Web_Template\Web_Template.csproj
dotnet user-secrets set "Authentication:Google:ClientSecret" "YOUR_CLIENT_SECRET" --project .\Web_Template\Web_Template.csproj
```

For production, store secrets in the hosting platform environment variables or secret manager.

## Example Program.cs Pattern

This is a learning example, not a drop-in replacement for the current project:

```csharp
builder.Services
    .AddAuthentication()
    .AddGoogle(options =>
    {
        options.ClientId = builder.Configuration["Authentication:Google:ClientId"]!;
        options.ClientSecret = builder.Configuration["Authentication:Google:ClientSecret"]!;
    });
```

If you use ASP.NET Core Identity, follow the official Identity setup so cookies, users, roles, password reset, and external login linking are handled correctly.

## Other Login Providers

The same idea applies to other providers:

- Microsoft account or Entra ID
- GitHub
- Discord
- Email and password
- Magic link email login

Use official provider documentation, create a developer app, set redirect URIs, store secrets safely, and connect the provider through server-side authentication code.

ASP.NET Core external provider overview:

[Using external login providers with Identity in ASP.NET Core](https://learn.microsoft.com/en-us/aspnet/core/security/authentication/social/)

## Replace The Demo Buttons

In `Login.cshtml`, find:

```html
<button class="provider-button google-provider" type="button" data-auth-demo="Google">
```

After real authentication exists, replace the demo button with a form or link that starts the external login challenge.

Do not make a front-end-only login system. A real login needs server-side validation, secure cookies, CSRF protection, account linking, logout, and secret storage.

## GitHub Pages Reminder

Use GitHub Pages to preview the design. Use an ASP.NET Core hosting service when you are ready for real login, database, sessions, and user accounts.
